# ⚡ 실전 DDoS 방어 테스트 및 검증 가이드

작성일: 2026.06.11

본 문서는 Sketch-Gate 시스템이 멀티스레드 환경에서 분당 트래픽 제한(Rate Limit)을 정확히 수행하고, 악성 공격자를 실시간으로 칼차단하는지 검증하기 위한 실전 시뮬레이션 가이드입니다.

---

<br/>

### 1. 사전 준비 (서버 구동)
프로젝트 루트 디렉터리에서 Netty 엔진 서버를 구동합니다. 
* **현재 설정**: 기본 8080 포트, 분당 300회 제한
* **설정 파일**: `app/src/main/resources/sketch_gate.properties`

    ```cmd
    .\gradlew run
    ```

<br/>

### 2. 시뮬레이션 시나리오 및 테스트 명령어
- **🌐 시나리오 A**: 정상 유저의 일상적인 접근
새로운 터미널(CMD) 창을 열고 아래 명령어를 실행합니다.

    ```cmd
    curl http://localhost:8080
    ```

    기대 결과: 서버가 정상적으로 요청을 수락하며, 화면에 templates/welcome.html에 정의된 200 OK 정상 페이지가 출력됩니다.
    
    <br/>

- **⚡ 시나리오 B**: 자동화 봇을 이용한 무차별 DDoS 폭격 (300회 초과 난사)
새로운 터미널 창에서 아래의 Windows 루프 명령어를 복사하여 붙여넣은 후 엔터를 누릅니다. (안전하게 임계치를 넘기기 위해 빠른 속도로 500회 연속 curl을 요청하는 스크립트입니다.)

    ```cmd
    for /L %i in (1,1,500) do @curl -s http://localhost:8080
    ```

<br/>

### 3. 성공 기준 및 검증 포인트
- 🧑‍💻 클라이언트(공격자) 측 화면:

    처음 300회까지는 정상 페이지가 출력되다가, 정확히 301번째 요청이 도달하는 나노초 타이밍부터 아래의 차단 응답이 사정없이 쏟아져야 성공입니다.
    
    ```text
    403 Forbidden: DDoS Attack Detected.
    ```

<br/>

- 🖥️ 서버(Gateway) 콘솔 창:

    수많은 폭격 중 오직 최초로 임계치를 넘은 순간 딱 1번만 아래의 강렬한 경고 로그가 찍히며, 이후 요청은 연산 오버헤드 없이 최전방에서 조용히 드롭(Silent Drop)되어 서버 자원을 보호하는지 확인합니다.

    ```text
    --------------------------------------------------
    [WARN] ALERT! 임계치 초과 공격 IP 즉시 차단: 127.0.0.1
    [WARN] 현재 분당 인입량 추정치: 301 (제한: 300)
    --------------------------------------------------
    ```
